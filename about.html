MP4 to MP3 || MP3 to Spotify

About A/V to Spotify?

A/V to Spotify was created out of a desire to be able to save unreleased music from your favorite artists onto your spotify account, so it could be in the same place as all your other music. It is a program that does 2 things:
1. Converts your Youtube videos and other MP4 media files into an MP3 files, which you can download and save onto your computer.
2. Uploads your newly created or any other MP3 files you have saved onto your spotify account.

How it works: MP4 to MP3

Once you drag and drop or select your MP4 file, A/V to Spotify will use different python libraries and frameworks to convert it to MP3. Similarly, when you paste your Youtube link into the interactive bar, it will go through its technologies, and spit out your MP3 file.

Technology Usage More Info
FFMPEG FFMPEG is a multimedia framework that can perform a number of computational transformations on MP3 and MP4 data. FFMPEG, accompanied with subprocess, is used to convert the MP4 files to MP3, later downloading it onto your desktop. link
Subprocess Subprocess module is a tool that allows you to run other programs or commands from your Python code. It can be used to open new programs, send them data and get results back. In A/V to Spotify, it connects with FFMPEG to directly convert files in the command line, expoditing the process. link
PYtube PYtube is a dependence free library that can download Youtube videos, convert them to MP3 files, and perform other manipulations on YouTube files. Simply put, A/V to Spotify takes the Youtube link, runs PYtube commands on said link, and converts and downloads an MP3 file. link

How it works: MP3 to Spotify

Once you select or drag and drop your MP3 file, the file goes through Spotify's API, which transfers your MP3 data directly to your Spotify account. Once it's uploaded, you can listen to your new music!
Technologies used:

Technology Usage More Info
Spotify API Spotify API takes the data you want to upload to the user's spotify and passes it through to that user's account. Once it's on Spotify's end, the file name, image and other interface changes can be made. link
JavaScript JavaScript was used to add functionality to the site, and make it interactive. While it is less crucial of a technology than the Spotify API, it is still important to the functionality of the site. link
MongoDB MongoDB was used to create the backend of the program. In the backend, there are files for the MP4 to MP3 part of the program, as well as a file/files to implement the token and API for Spotify. It is essential to the program. link
